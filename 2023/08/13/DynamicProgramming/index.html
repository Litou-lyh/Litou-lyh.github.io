1.1 Definition

Dynamic Programming is a technique in computer programming that helps to efficiently solve a class of problems that have overlapping subproblems and optimal substructure property.

Step 1: Define the dp array

The meaning of the values in the array is typically defined as the goal of the problem, but the coordinates can vary according to the induction steps.

Step 1: Come up with induction steps

$$
dp[\vec{v}_{k+1}] = f(dp[\vec{v}k], dp[\vec{v}{k-1}], â€¦, dp[\vec{v}_0])
$$

where $dp[\cdot]$ represents the dp array, subscript $k$ means the k-th calculated sub-problem. $\vec{v}_{k}$ denotes indexes / coordinates of sub-problems. The coordinate vector $\vec{v}$ are typically 1 / 2 dimensions. $f$ can be max, min, sum, avg, minmax, max(sum()) etc.

Step 2: Consider the base cases

Base cases can be either a single case, 1-d array, 2 x 1-d array, or 2-d array.

Step 3: Consider the pipeline

  1. Top-down (Memoization)

    Starting from recursive methods, memorize any computed result, if the visited sub-problem has been calculated, just retrieve.

  2. Bottom-up (Induction)

    Do for loop from base cases and induct the next sub-problem. Not applicable for all problems.

3.2 Longest Common Subsequence

Input : 2 Sequences of length $m$ & $n$ ($X[1..m]$, $Y[1..n]$)

Output: length of $LCS(X, Y)$

Sequences LCS Length
S P R I ==N== ==G== T ==I== M ==E== / /
P I O N E E R PINE 4
E ==N== ==G== ==I== N ==E== NGIE 4

Induction

â€‹ Consider $X[m]$ and $Y[n]$:

â€‹ (a) If $X[m] \neq Y[n]$, either $X[m] \notin LCS$ or $Y[n] \notin LCS$,

â€‹ $LCS[m, n] = \max (LCS[m-1, n], LCS[m, n-1])$

â€‹ (b) If $X[m] = Y[n]$, then $X[m] = Y[n]$ must be part of $LCS$

â€‹ $LCS[m, n] = LCS[m-1, n-1] + 1$

Base Case

$LCS[0, n] = LCS[m, 0] = 0$

Pipeline

Both
