Realtime Class Notes

1 CAP Theorem

C: Consistency, After a write operation, any access from any users to any nodes returns the same data.

A: Availability. Any request returns a feasible response (not an error caution) in an acceptable period of time.

P: Partition Tolerance. System’s ability to continue functioning despite partitions.

但是，这道 CAP 的选择题，就像别人在问你“小明的父亲有三个孩子，老大叫大朗，老二叫二郎，请问老三叫什么”一样。在以分布式存系统为限定条件的 CAP 世界里，P 是早已经确定的答案，P 是必须的。

因为，在分布式系统内，P 是必然的发生的，不选 P，一旦发生分区错误，整个分布式系统就完全无法使用了，这是不符合实际需要的。所以，对于分布式系统，我们只能能考虑当发生分区错误时，如何选择一致性和可用性。

Thus, P is satisfied by default in distributed systems. The only two options are C&P and A&P.

Example 1:

To ensure the consistency of the system upon the disconnection (partition) between Node 1 and Node 2, the system has to return errors and refuses to return any data. This is an example of (C & P).

Example 2:

If the system chooses to provide high-level availability, the system must be able to return feasible data within a short period even if the connections are down or some nodes are down (partition). As a trade-off, the consistency is abandoned, that is, every request can retrieve different versions of data.
